# 战斗技能系统 — 详细迭代方案

## 一、系统总览

玩家在**藏经阁**浏览并学习战斗技能，在**技能界面**装卸已学技能，装备的技能在**回合制战斗**中生效。技能受「记忆点数」限制，记忆点上限由**学识**属性决定。

### 核心概念

| 概念 | 说明 |
|------|------|
| **记忆点上限** | 由学识换算，0学识→0点，300学识→10点 |
| **技能等级** | 每个技能 Lv1~Lv5，逐级解锁 |
| **学习** | 消耗金钱，满足前置条件后永久获得该等级 |
| **装备** | 把已学技能放入「记忆槽」，占用记忆点数 |
| **生效** | 只有已装备的技能在战斗中触发 |

## 二、数据结构设计

### 2.1 技能定义（静态配置，放 `module/skill-list.js`）

#### 2.1.1 triggerTiming 候选值（可扩展）

| triggerTiming | 含义 | 示例效果 |
|---------------|------|---------|
| `onTauntHitDefend` | 嘲讽命中防御时 | 额外伤害 |
| `onAttackHit` | 任意攻击命中时 | 额外吸血/穿甲 |
| `onDefend` | 选择防御时 | 额外回血/反伤 |
| `onGather` | 选择集气时 | 额外获得能量 |
| `onTakeDamage` | 受到伤害时 | 减伤/反击 |
| `onSpecialUse` | 使用绝招时 | 绝招增伤 |
| `onBattleStart` | 战斗开始时 | 永久 buff |
| `onCrit` | 暴击触发时 | 额外效果 |
| `passive` | 被动常驻 | 属性加成 |

> 每新增一种 `triggerTiming`，只需在 `turn-based-battle-new.html` 的对应位置加一个 `applySkillEffect(timing, context)` 调用即可。

## 三、记忆点数公式

### 3.1 公式设计

仿照 `calculateLevelFromWuxue` 的递增累计制：

```
第 i 个记忆点需要的学识 = 8 + 4i （i = 1, 2, ..., 10）
```

| 记忆点 | 本级所需学识 | 累计学识 |
|--------|------------|---------|
| 1 | 12 | 12 |
| 2 | 16 | 28 |
| 3 | 20 | 48 |
| 4 | 24 | 72 |
| 5 | 28 | 100 |
| 6 | 32 | 132 |
| 7 | 36 | 168 |
| 8 | 40 | 208 |
| 9 | 44 | 252 |
| 10 | 48 | 300 |

恰好 0 学识 → 0 点，300 学识 → 10 点。

### 3.2 实现函数（`game-utils.js`）

```js
function calculateMemorySlots(xueshi) {
    let totalCost = 0;
    let slots = 0;
    for (let i = 1; i <= 10; i++) {
        totalCost += 8 + 4 * i;
        if (xueshi >= totalCost) {
            slots = i;
        } else {
            break;
        }
    }
    return slots;
}

function getUsedMemorySlots() {
    let used = 0;
    for (const [skillId, level] of Object.entries(equippedSkills)) {
        const skill = skillList[skillId];
        if (skill) {
            used += skill.levels[level - 1].memorySlots;
        }
    }
    return used;
}
```

## 五、战斗集成

### 5.1 数据传递

发起战斗时（`showBattleGame`），将已装备技能序列化为 URL 参数：

```js
// 在 showBattleGame 中追加参数
const equippedSkillsData = {};
for (const [skillId, level] of Object.entries(equippedSkills)) {
    const skill = skillList[skillId];
    if (skill) {
        equippedSkillsData[skillId] = {
            triggerTiming: skill.triggerTiming,
            params: skill.levels[level - 1].params
        };
    }
}
params.append('equippedSkills', JSON.stringify(equippedSkillsData));
```

### 5.2 战斗页面接收与处理

在 `turn-based-battle-new.html` 中：

```js
// 解析技能数据
const equippedSkills = JSON.parse(
    new URLSearchParams(location.search).get('equippedSkills') || '{}'
);

// 通用技能触发函数
function applySkillEffects(timing, context) {
    // context 包含当前战斗状态：attacker, defender, damage, action 等
    let result = { extraDamage: 0, extraHeal: 0, extraEnergy: 0, message: '' };
    
    for (const [skillId, data] of Object.entries(equippedSkills)) {
        if (data.triggerTiming === timing) {
            // 按 triggerTiming 分发处理
            switch (timing) {
                case 'onTauntHitDefend':
                    result.extraDamage += context.basicDamage * data.params.extraDamageRatio;
                    result.message += `【${skillList[skillId]?.name}】发动！`;
                    break;
                // ... 其他 timing 的处理
            }
        }
    }
    return result;
}
```

### 5.3 挂钩位置示例：嘲讽命中防御

在 `executeActions()` 中处理嘲讽对防御的逻辑处（约 turn-based-battle-new.html #L3600 附近）：

```js
// 原有逻辑：对方进入挑衅状态 + 气-1
// 新增：
if (isPlayer) {  // 仅玩家技能生效
    const skillResult = applySkillEffects('onTauntHitDefend', {
        basicDamage: PLAYER_INFO.basicDamage,
        attacker: PLAYER_INFO,
        defender: ENEMY_INFO
    });
    if (skillResult.extraDamage > 0) {
        ENEMY_INFO.currentHealth -= skillResult.extraDamage;
        // 显示技能触发日志
        addBattleLog(skillResult.message + ` 造成 ${Math.round(skillResult.extraDamage)} 点额外伤害！`);
    }
}
```

## 八、待讨论 / 可选项

以下问题可以先推进再调整，也可以现在确认：

1. **技能是否占用行动点？** 学习技能是否需要消耗行动点（类似练武/学习），还是纯金钱购买？
   - 当前方案：纯金钱购买，不消耗行动点。

2. **技能遗忘/重置？** 是否允许遗忘已学技能（退还部分金钱），还是学了就永久拥有？
   - 当前方案：永久拥有，不可遗忘。

3. **NPC切磋是否也受技能影响？** 目前切磋也走 `turn-based-battle-new.html`。
   - 当前方案：是的，装备的技能在所有战斗中生效。

4. **敌人是否也有技能？** 后续是否需要给敌人也配技能？
   - 当前方案：暂不实现，敌人不使用技能。后续可在 `battleData.enemy` 中追加 `skills` 字段。

5. **初始技能数量？** 第一版先做几个技能？
   - 建议：先做 3~5 个覆盖不同 `triggerTiming` 的技能用于验证框架，后续随时可在 `skill-list.js` 中拓展。

6. **技能与门派武功 `martialArts` 的关系？** 现有 `martialArts`（太白仙迹等）目前看起来是占位的，是否要合并或独立？
   - 当前方案：独立系统，互不影响。`martialArts` 可以后续再规划。
