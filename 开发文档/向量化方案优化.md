# LLM 角色扮演游戏历史记录管理方案

> **版本**：v1.0
> **日期**：2026-06-09
> **基础模型**：`BAAI/bge-m3`
> **目标**：构建一套全自动、用户无感的角色扮演游戏长期记忆管理系统

---

## 一、方案总览

#### 1.1 核心设计理念

| 设计原则 | 说明 |
|---------|------|
| **摘要索引、全文返回** | 用 SUMMARY 做向量化（语义提纯），召回后取出对应 MAIN_TEXT 喂给 LLM |
| **双路向量、混合检索** | bge-m3 同时输出 dense + sparse，分别召回再融合 |
| **双查询并行** | 用户输入 + 上一场 SUMMARY 各自构建查询，结果层 RRF 融合 |
| **元数据硬过滤** | NPC、地点、时间作为前置过滤条件，缩小候选集 |
| **零人工干预** | 全程自动切分、向量化、召回、融合，无需手动标注 |

#### 1.2 系统架构图


#### 2.1 数据结构定义

每一场戏（scene）入库为一条记录：

```python
scene_record = {
```

#### 2.2 向量化代码

```python
from FlagEmbedding import BGEM3FlagModel

model = BGEM3FlagModel('BAAI/bge-m3', use_fp16=True)

def vectorize_scene(scene_data: dict) -> dict:
    """
    将一场戏的 SUMMARY 向量化，返回入库记录。
    """
    summary = scene_data["summary"]

    # 一次 encode，双路输出
    output = model.encode(
        summary,
        return_dense=True,
        return_sparse=True,
        return_colbert_vecs=False,    # 暂不启用 colbert
        max_length=512,
    )

    return {
        "scene_id":    scene_data["scene_id"],
        "scene_index": scene_data["scene_index"],

        "dense":  output['dense_vecs'].tolist(),
        "sparse": {
            int(k): float(v)
            for k, v in output['lexical_weights'].items()
        },

        "metadata": parse_side_note(scene_data["side_note"]),
        "payload": {
            "main_text": scene_data["main_text"],
            "summary":   summary,
            "side_note": scene_data["side_note"],
        }
    }
```

#### 2.3 入库流程

```python
def ingest_scene(llm_response: str, scene_index: int):
    """
    LLM 完成一次回复后调用，自动入库。
    """
    # 1. 解析 LLM 输出（提取 MAIN_TEXT / SUMMARY / SIDE_NOTE）
    scene_data = parse_llm_response(llm_response, scene_index)

    # 2. 向量化
    record = vectorize_scene(scene_data)

    # 3. 写入向量库（推荐 Milvus 2.4+）
    vector_db.insert(record)

    return record["scene_id"]
```

## 三、召回阶段：双查询并行混合检索

#### 3.1 查询向量构建

```python
def build_dual_queries(user_input: dict, last_summary: str):
    """
    构建双查询：当前意图 + 历史承接。
    """
    # ===== Q_intent: 用户当前意图 =====
    intent_text = extract_intent_text(user_input)

    q_intent = model.encode(
        intent_text,
        return_dense=True,
        return_sparse=True,
        max_length=512,
    )

    # ===== Q_context: 上一场承接 =====
    q_context = model.encode(
        last_summary,
        return_dense=True,
        return_sparse=True,
        max_length=512,
    )

    return q_intent, q_context


def extract_intent_text(user_input: dict) -> str:
    """
    从用户结构化输入中抽取语义文本，丢弃骨架字段。
    """
    parts = []

    if user_input.get("事件描述"):
        parts.append(user_input["事件描述"])
    if user_input.get("行动选择"):
        parts.append(f"高欢选择{user_input['行动选择']}")
    if user_input.get("选择结果"):
        parts.append(f"结果：{user_input['选择结果']}")
    if user_input.get("特殊事件"):
        parts.append(user_input["特殊事件"])
    if user_input.get("抵达目的地"):
        parts.append(f"抵达{user_input['抵达目的地']}")
    if user_input.get("在场NPC"):
        npcs = user_input["在场NPC"]
        if isinstance(npcs, list):
            npcs = "、".join(npcs)
        parts.append(f"在场角色：{npcs}")
    if user_input.get("地点"):
        parts.append(f"地点：{user_input['地点']}")

    return "。".join(parts) if parts else "继续推进剧情"
```

#### 3.2 元数据硬过滤

```python
def build_metadata_filter(user_input: dict) -> dict:
    """
    构建向量库的前置过滤条件。
    """
    filters = {}

    # NPC 过滤：召回历史上和当前在场 NPC 有过互动的场景
    npcs = user_input.get("在场NPC") or user_input.get("随行NPC")
    if npcs:
        if isinstance(npcs, str):
            npcs = [npcs]
        filters["npcs_overlap"] = npcs   # 任一交集即可

    # 地点过滤：可选，根据业务决定是否启用
    # location = user_input.get("地点")
    # if location:
    #     filters["location"] = location

    # 时间窗口：优先召回最近 N 周的剧情
    if user_input.get("时间"):
        filters["time_window_recent"] = 8  # 最近 8 周

    return filters
```

> **注意**：元数据过滤建议**软过滤**——先尝试带过滤召回，如果结果不足 K 条再放宽条件二次召回，避免主线伏笔被过早排除。

#### 3.3 双查询 × 双路向量 = 四路并行召回

```python
def hybrid_recall(user_input: dict, last_summary: str, top_k: int = 5):
    """
    全自动混合检索主入口。
    """
    # 1. 构建查询向量
    q_intent, q_context = build_dual_queries(user_input, last_summary)

    # 2. 元数据过滤
    metadata_filter = build_metadata_filter(user_input)

    # 3. 四路并行召回（每路各取 20 条）
    K_RECALL = 20

    hits = {
        ("intent",  "dense"):  vector_db.dense_search(
            q_intent['dense_vecs'],
            filter=metadata_filter,
            k=K_RECALL
        ),
        ("intent",  "sparse"): vector_db.sparse_search(
            q_intent['lexical_weights'],
            filter=metadata_filter,
            k=K_RECALL
        ),
        ("context", "dense"):  vector_db.dense_search(
            q_context['dense_vecs'],
            filter=metadata_filter,
            k=K_RECALL
        ),
        ("context", "sparse"): vector_db.sparse_search(
            q_context['lexical_weights'],
            filter=metadata_filter,
            k=K_RECALL
        ),
    }

    # 4. RRF 融合
    fused_results = rrf_fusion(hits, RRF_WEIGHTS, k_constant=60)

    # 5. 时间衰减加权（可选）
    final_results = apply_time_decay(fused_results, user_input)

    # 6. 取 Top-K，返回 MAIN_TEXT
    top_scenes = final_results[:top_k]
    return [vector_db.get_payload(sid)["main_text"] for sid, _ in top_scenes]
```

#### 3.4 RRF 融合算法

```python
# ===== 推荐权重配置 =====
RRF_WEIGHTS = {
    # (查询路, 向量路): 权重
    ("intent",  "dense"):  1.00,    # 用户意图 + 语义匹配（主力）
    ("intent",  "sparse"): 0.85,    # 用户意图 + 关键词匹配（专名命中）
    ("context", "dense"):  0.70,    # 历史承接 + 语义匹配
    ("context", "sparse"): 0.55,    # 历史承接 + 关键词匹配
}


def rrf_fusion(hits: dict, weights: dict, k_constant: int = 60) -> list:
    """
    Reciprocal Rank Fusion: score = Σ weight / (k + rank)

    Args:
        hits: {(query_type, vec_type): [hit_list]}
        weights: 每路的融合权重
        k_constant: RRF 平滑常数（业界默认 60）

    Returns:
        [(scene_id, score), ...] 按得分降序
    """
    rrf_scores = {}

    for route_key, hit_list in hits.items():
        weight = weights.get(route_key, 1.0)

        for rank, hit in enumerate(hit_list):
            sid = hit.scene_id
            rrf_scores[sid] = (
                rrf_scores.get(sid, 0.0)
                + weight / (k_constant + rank)
            )

    return sorted(rrf_scores.items(), key=lambda x: -x[1])
```

#### 3.5 权重设计说明

##### 入库阶段权重（dense vs sparse）

入库时**两路向量平等存储**，无需权重——权重只在召回融合时体现。

##### 召回阶段权重（4 路 RRF）

| 路径 | 权重 | 设计理由 |
|------|-----|---------|
| **intent × dense** | **1.00** | 用户当前意图 + 语义匹配，最直接相关，作为主基线 |
| **intent × sparse** | **0.85** | 用户输入中的专有名词（NPC、地名、道具）精确命中，对剧情场景至关重要 |
| **context × dense** | **0.70** | 上一场摘要承接，提供故事线连续性，但权重低于当前意图避免主导 |
| **context × sparse** | **0.55** | 历史关键词匹配，作为兜底信号 |

**权重设计原则**：

1. **intent 整体 > context 整体**（1.85 vs 1.25）
   保证用户当前意图主导召回，避免被无关历史拖偏。
2. **dense 整体 ≈ sparse 整体**（1.70 vs 1.40）
   语义匹配略高于关键词匹配，但差距不大——剧情场景中专名匹配价值很高。
3. **两端权重比 ≈ 1.8 : 1**
   即使 context 路全部命中、intent 路全部不命中（极端情况），intent 仍有翻盘空间。

##### 自适应能力

这套权重对**四种典型输入密度**都能自动适应：

| 输入类型 | 行为表现 |
|---------|---------|
| **高密度（输入1）** | intent 路召回质量高，4 路中 3 路被它主导，结果以当前剧情为中心 ✅ |
| **中密度（输入2/3）** | intent 和 context 都贡献信号，融合后获得平衡的相关历史 ✅ |
| **低密度（输入4）** | intent 路召回零散，context 路兜底，避免空召回 ✅ |

**无需 if-else 判断输入类型**，权重设计本身就是鲁棒的。

#### 3.6 时间衰减（可选增强）

```python
import math

def apply_time_decay(
    fused_results: list,
    user_input: dict,
    decay_lambda: float = 0.05
) -> list:
    """
    给最终得分叠加时间衰减因子，让近期场景排名更靠前。
    """
    current_index = user_input.get("current_scene_index", 0)
    adjusted = []

    for sid, score in fused_results:
        scene_index = vector_db.get_metadata(sid)["scene_index"]
        delta = current_index - scene_index
        decay = math.exp(-decay_lambda * delta)
        adjusted.append((sid, score * decay))

    return sorted(adjusted, key=lambda x: -x[1])
```

> **λ = 0.05** 大致表示 14 场之前的戏权重衰减到 50%。可根据用户体验微调。

```python
def chat_turn(user_input: dict) -> str:
    """
    一次完整的用户对话回合。

    llm_response = llm.generate(context)

    # 4. 入库（自动向量化）
    new_scene_id = ingest_scene(
        llm_response,
        scene_index=get_next_scene_index()
    )

    # 5. 更新会话状态（保存最新 SUMMARY 作为下一轮的 context）
    update_last_summary(extract_summary(llm_response))

    return llm_response
```

## 八、关键设计决策回顾

| 决策点 | 选择 | 理由 |
|-------|-----|------|
| 索引粒度 | SUMMARY（非 MAIN_TEXT）| 短文本语义提纯，避免向量被稀释 |
| 向量类型 | dense + sparse（暂不上 colbert）| 性价比最优，覆盖 80% 收益，存储成本仅为完整方案 1/100 |
| 查询构建 | 双查询并行（非加权求和）| 避免向量空间语义漂移，可调试性强 |
| 融合算法 | RRF（非分数加权和）| 对分数尺度不敏感，鲁棒性高，工业标准 |
| 元数据过滤 | 软过滤前置 | 缩小候选集，提升精度同时不丢失主线伏笔 |
| 自动化程度 | 全自动，无人工切分 | bge-m3 原生支持，零运维负担 |

## 九、一句话核心心法

> **"摘要做索引、全文做载荷；意图找当下、上下文承前情；双路抓语义和专名、RRF 融合不漂移；元数据来兜底、时间衰减保新鲜。"**

整套方案对玩家完全无感——他们只感受到 NPC 越来越懂自己、剧情伏笔总在恰当时机回响、世界仿佛真的在记得每一段过往。这正是优秀角色扮演体验的核心：**记忆不是数据结构，而是叙事的灵魂** 🎯
